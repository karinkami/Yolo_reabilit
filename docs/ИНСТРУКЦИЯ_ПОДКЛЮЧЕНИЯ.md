# Инструкция по подключению проекта YOLO Reabilit

---

## 1. Что установить

| Компонент | Версия | Зачем |
|-----------|--------|--------|
| **Python** | 3.11+ | Сервер Flask, YOLO |
| **Git** | актуальная | Скачать репозиторий |
| **PostgreSQL + pgAdmin** | 15+ | База данных (обязательно) |
| **Node.js** | 20+ | Сборка React (`npm run build`) |

---

## 2. Скачать проект с GitHub

```powershell
git clone https://github.com/ВАШ_АККАУНТ/Yolo_reabilit.git
cd Yolo_reabilit
```

---

## 3. Виртуальное окружение Python

В корне проекта (где лежит `run.py`):

```powershell
python -m venv yolo_env
.\yolo_env\Scripts\Activate.ps1
pip install --upgrade pip
pip install -r requirements.txt
```

Если PowerShell блокирует скрипты (один раз):

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

**Альтернатива (cmd):** `yolo_env\Scripts\activate.bat`

В начале строки должно появиться `(yolo_env)` — все команды `pip` и `python` дальше только с активированным окружением.

Файл `requirements.txt` сохраняйте в **UTF-8** (не «Unicode» в Блокноте).

При первом запуске Ultralytics скачает веса pose-модели (например `yolo11n-pose.pt`) — нужен интернет.

## 4. База данных PostgreSQL

Приложение работает **только с PostgreSQL**. Файл `appsettings.json` с строкой подключения **обязателен** (или переменная окружения `DATABASE_URI`).

### Что создаётся автоматически при запуске

При каждом `python run.py` вызывается `create_app()` (`app/__init__.py`), который:

- создаёт таблицы, если их ещё нет (`db.create_all()`);
- подтягивает недостающие колонки в старой БД (`schema_patch`);
- заполняет каталог упражнений и при необходимости демо-врача (сид).

**Пустую базу PostgreSQL** приложение **не** создаёт — только таблицы **внутри** уже существующей базы `yolo_rehab`.

| Уровень | Что это | Кто создаёт |
|---------|---------|-------------|
| **Сервер БД** | PostgreSQL: роль + база `yolo_rehab` | **Вы** (SQL в pgAdmin) |
| **Таблицы приложения** | `users`, `exercises`, `training_sessions` и др. | **Автоматически** при `run.py` **или** скриптом `init_database.py` |

---

### Шаг 1. Создать пользователя (роль)

1. Откройте **pgAdmin**, войдите под **`postgres`**.
2. **Tools → Query Tool** (база `postgres`).
3. Откройте файл **`scripts/create_postgres_01_role.sql`**.
4. **Замените пароль** в `CREATE ROLE ... PASSWORD '...'` на свой.
5. Выполните (**F5**).

### Шаг 2. Создать пустую базу данных

**Через интерфейс pgAdmin:**

1. Правый клик **Databases → Create → Database**.
2. **Database:** `yolo_rehab`
3. **Owner:** `yolo_rehab`
4. **Encoding:** UTF8  
5. Сохранить.

**Или SQL** — откройте **`scripts/create_postgres_02_database.sql`**, выполните **только** команду `CREATE DATABASE` в **новом** Query Tool (не в одной транзакции с другими командами).

### Шаг 3. Права на схему (PostgreSQL 15+)

1. В дереве pgAdmin **выберите базу `yolo_rehab`**.
2. **Tools → Query Tool** (запросы идут **в этой** базе).
3. Выполните **`scripts/grant_schema_yolo_rehab.sql`** целиком (**F5**).

### Шаг 4. Подключить проект к базе

1. Скопируйте **`appsettings.example.json`** → **`appsettings.json`** в корне проекта.
2. Укажите строку подключения (подставьте свой пароль):

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "postgresql+psycopg2://yolo_rehab:ВАШ_ПАРОЛЬ@localhost:5432/yolo_rehab"
  },
  "SecretKey": "длинная-случайная-латиница-без-кириллицы"
}
```

Формат URI:

```text
postgresql+psycopg2://ИМЯ_РОЛИ:ПАРОЛЬ@ХОСТ:ПОРТ/ИМЯ_БАЗЫ
```

Пароль без пробелов; спецсимволы `@`, `#`, `:` — URL-кодировать или использовать латиницу и цифры.

**Приоритет** (если нужно переопределить без файла):

1. переменная окружения `DATABASE_URI`
2. `appsettings.json`

### Шаг 5. Создание таблиц и начальных данных (сид)

После настройки Postgres:

#### Способ 1 — вручную (рекомендуется перед первым запуском)

Из корня проекта, с активированным `yolo_env`:

```powershell
python scripts\init_database.py
```

**Или** обёртка для Windows (сама найдёт Python в `yolo_env`):

```powershell
.\scripts\create_database.ps1
```

Скрипт выведет, к какой БД подключился, и сколько упражнений/врачей в каталоге. Сайт при этом **не** запускается.

#### Способ 2 — автоматически при запуске сайта

```powershell
python run.py
```

Те же действия выполняются внутри `create_app()`:

- `db.create_all()` — все таблицы по моделям SQLAlchemy;
- `schema_patch` — недостающие колонки в старых БД;
- `seed_exercises()` — 10 упражнений в каталоге;
- `seed_demo_doctor()` — демо-врач (если ещё нет врача в БД).

Оба способа **безопасно повторять**: существующие данные пациентов и тренировок не стираются, сид только дополняет каталог и создаёт врача, если его не было.

**Демо-врач по умолчанию:**

| Поле | Значение |
|------|----------|
| Email | `doctor@clinic.local` |
| Пароль | `doctor123` |

Переопределение до первого запуска: `SEED_DOCTOR_EMAIL`, `SEED_DOCTOR_PASSWORD`.

Пациенты регистрируются через форму на сайте.

### Опционально: тексты упражнений в БД

**После** шага 5 (когда таблица `exercises` уже заполнена):

| Файл | Действие |
|------|----------|
| `scripts/exercise_guides_schema_and_data.sql` | В pgAdmin, база `yolo_rehab`, выполнить целиком |

Без этого скрипта инструкции к упражнениям всё равно работают — из файла `app/exercise_guides_defaults.py`.

## 5. Сборка интерфейса (обязательно)

```powershell
cd frontend
npm install
npm run build
cd ..
```

Результат попадает в **`app/static/web-dist/`**. Без этой папки сайт отдаёт **503** («интерфейс не собран»).

Разработка UI отдельно: `cd frontend` → `npm run dev` (порт 5173, прокси на Flask `http://127.0.0.1:5000`).

---

## 6. Запуск приложения

```powershell
python run.py
```

Откройте в браузере адрес из консоли (обычно `http://127.0.0.1:5000`).

**Тренировка с камерой (Windows):** в `run.py` релоадер Flask по умолчанию **выключен**, иначе второй процесс часто блокирует веб-камеру. Включать только при отладке бэкенда:

```powershell
$env:FLASK_USE_RELOADER = "1"
python run.py
```

После правок React-интерфейса снова выполните `npm run build` в `frontend/` и обновите страницу (**Ctrl+F5**).

Войти: демо-врач или зарегистрированный пациент.

| Роль | URL (после входа) |
|------|-------------------|
| Врач | `/doctor/` |
| Пациент, тренировка | `/patient/rehab/` |
| Пациент, статистика | `/patient/stats/` |

## 7. Список скриптов в `scripts/`

| Файл | Когда выполнять |
|------|-----------------|
| `init_database.py` | **Вручную:** таблицы + сид без веб-сервера |
| `create_database.ps1` | То же для Windows (вызывает `init_database.py`) |
| `create_postgres_01_role.sql` | Один раз: роль Postgres |
| `create_postgres_02_database.sql` | Один раз: база `yolo_rehab` |
| `grant_schema_yolo_rehab.sql` | Один раз: права в базе `yolo_rehab` |
| `exercise_guides_schema_and_data.sql` | По желанию: гайды в БД (синхрон с `exercise_guides_defaults.py`) |
| `export_exercise_guides_sql.py` | Пересобрать SQL после правки текстов в `app/exercise_guides_defaults.py` |
